# Plan: Multi-Agent Flowchart Architecture

## Context

The current flowchart agent is a single `LlmAgent` doing everything — asking questions, validating answers, saving state, handling commands. The user wants to split this into a proper multi-agent hierarchy:

1. **Principal Agent (PA)** — orchestrator managing conversation flow
2. **Question Tracker Agent** — determines and presents the next question
3. **Answer Validator Agent** — validates answers before saving, then advances to next question

ADK's `LlmAgent` natively supports `sub_agents` with automatic `transfer_to_agent` tool injection. Sub-agents share the same `session.state` and conversation history. This is the right pattern (NOT `AgentTool` which creates isolated sessions, and NOT `LoopAgent` which doesn't pause for user input).

## Architecture

```
root_agent (PA) — LlmAgent
  ├── question_tracker_agent — LlmAgent (sub_agent)
  └── answer_validator_agent — LlmAgent (sub_agent)
```

**Agent responsibilities and tool distribution:**

| Agent | Role | Tools |
|-------|------|-------|
| PA (`flowchart_agent`) | Orchestrator: routes to sub-agents, handles special commands | `load_user_history`, `load_prefilled_answers`, `check_assessment_status`, `restart_assessment` |
| Question Tracker | Determines next question, presents it naturally | `get_next_question` |
| Answer Validator | Validates answer type, saves if valid, asks to retry if not | `save_user_answer`, `get_current_question_details` (NEW) |

**Conversation flow:**
- First message → PA calls `load_user_history` → transfers to Question Tracker
- Question Tracker calls `get_next_question` → presents question → waits
- User answers → PA transfers to Answer Validator
- Validator calls `get_current_question_details` → validates → calls `save_user_answer` → transfers to Question Tracker
- Invalid answer → Validator asks user to retry (stays as active agent)
- Special commands (status/restart/change answer) → PA handles directly
- Completion → Question Tracker reports completion, transfers back to PA
